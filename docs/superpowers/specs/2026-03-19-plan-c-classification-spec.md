# Plan-C 规格：分类/标签与查询

> 作用：仅用于分类标签能力实现计划输入

## 1. 目标

- 提供可编辑的用户分类体系与标签体系。
- 保证分类演进不破坏历史交易语义。
- 提供可组合的交易筛选能力。

## 2. 核心流程

1. 用户注册时复制默认分类模板。
2. 分类增删改查与归档管理。
3. 标签增删改查与交易关联。
4. 按账本/账户/分类/标签/时间多维筛选交易。

## 3. 接口范围

| Endpoint | Method | Auth | Success | Error |
|---|---|---|---|---|
| `/api/categories` | GET/POST/PUT/DELETE | Access/PAT | 分类结果 | `CAT_INVALID_PARENT`, `CAT_IN_USE_ARCHIVED` |
| `/api/tags` | GET/POST/PUT/DELETE | Access/PAT | 标签结果 | `TAG_DUPLICATED`, `TAG_NOT_FOUND` |

## 4. 数据与约束

- 分类结构仅支持一级 + 一级子分类。
- 已被交易引用的分类删除转归档，不做物理删除。
- 归档分类不可用于新建/编辑交易。
- 历史交易详情、统计、导出保留历史分类名称。

## 5. 异常与降级

- 非法父子关系：返回 `CAT_INVALID_PARENT`。
- 删除已引用分类：返回成功并附带“已归档”语义说明。
- 查询参数非法（时间反转、分页越界）：快速失败并返回参数错误。

## 6. 验收要点

- 分类归档后历史数据可追溯。
- 多条件筛选结果与明细一致。
- 查询性能阈值：1k 交易规模、分页 50 条、P95 <= 1 秒。

### Out of Scope

- 总资产口径计算、CSV 导入映射。
